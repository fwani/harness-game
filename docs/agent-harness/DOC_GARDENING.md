# Doc Gardening

> Generated by company-agent-harness v0.4.0.

## Why

문서가 오래되면 에이전트가 잘못 추측한다. 정기적으로 문서를 정리한다.

## When to update docs

- 동작/API/설정/운영 방식이 바뀔 때 같은 PR에서 문서를 갱신한다.
- 명령이 바뀌면 `.agent-harness.yml`과 `AGENTS.md`를 함께 갱신한다.

## Staleness signals (정리 기준)

- `> Status: Unknown` 또는 `_(unknown ...)_`가 오래 남아 있는 섹션
- 코드와 불일치하는 명령/경로
- 더 이상 존재하지 않는 파일을 가리키는 링크
- 6개월 이상 갱신되지 않은 핵심 문서

## Gardening loop

1. `scripts/agent-harness/agent-doc-check`로 필수 문서/섹션/링크 후보를 점검한다.
2. 오래된 항목을 정리하거나 갱신한다.
3. 큰 정리는 `docs/agent-harness/exec-plans/active/`에 계획으로 남긴다.
