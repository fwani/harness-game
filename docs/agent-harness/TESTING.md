## Policy

- 모든 코드 변경에는 테스트를 추가하거나 갱신한다.
- 새 기능: 동작을 검증하는 테스트를 추가한다.
- 버그 수정: 회귀를 막는 테스트를 먼저 추가한다.
- 검증을 실제로 실행하기 전에 "통과했다"고 말하지 않는다.
- 테스트 프레임워크가 없으면 사람에게 알리고 어떻게 할지 확인한다.

## How agents run tests

1. `scripts/agent-harness/agent-verify`를 실행한다.
2. 실패하면 출력을 그대로 보고하고 원인을 분석한다.
3. 수정 후 다시 검증한다.
