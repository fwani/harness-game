# Security

> Generated by company-agent-harness v0.4.0. 보안 규칙은 레포별 설정으로 약화할 수 없다.

## Hard rules

- 비밀정보, 토큰, 키, 자격증명을 커밋하지 않는다.
- 민감정보(`password`, `token`, `secret`, `authorization`)를 로그에 남기지 않는다.
- 인증/인가 흐름을 우회하거나 약화하지 않는다.
- 보안 판단이 필요한 경우 사람에게 에스컬레이션한다.

## Secret patterns to never commit

- `.env`, `.env.*`
- `*.pem`, `*.key`
- 토큰/자격증명이 포함된 설정 파일

`.agent-harness.yml`의 `security.secret_patterns`와 동기화한다.

## Destructive commands (사람 확인 전 실행 금지)

로컬 변경이나 데이터를 되돌릴 수 없게 만드는 명령은 사람 확인 전에 실행하지 않는다.
기본 차단 대상(`.agent-harness.yml`의 `security.destructive_commands`, 정규식):

- `git reset --hard`
- `git checkout -- .` / `git checkout -- <path>`
- `git clean -f`
- `git push --force` (단, `--force-with-lease`는 허용)
- `git branch -D`
- `rm -rf /` · `rm -rf ~` (절대/홈 경로 강제 삭제)
- 외부 리소스가 남아 있는데 DB만 삭제하는 등 정합성 깨는 삭제

### 기계적 가드 (선택, 권장)

`agent-harness guard`로 명령을 사전 검사할 수 있다(파괴적이면 exit 2):

```bash
agent-harness guard --cmd "git reset --hard"   # 차단
echo '{"tool_input":{"command":"git status"}}' | agent-harness guard  # 허용
```

Claude Code 등에서 PreToolUse 훅으로 연결하면 자동 차단된다(opt-in). 예: `.claude/settings.json`

```json
{
  "hooks": {
    "PreToolUse": [
      { "matcher": "Bash", "hooks": [ { "type": "command", "command": "agent-harness guard" } ] }
    ]
  }
}
```

## Escalate to human for

- 데이터 삭제
- 인증/인가 변경
- 권한 변경
- 프로덕션 마이그레이션
- 외부 고객 대상 API 변경
- 큰 인프라 비용 변경
