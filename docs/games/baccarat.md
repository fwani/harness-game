# 바카라 (Baccarat) 사양

> 규칙 + 구현 상태 + UI/UX 요구사항. 상위 인덱스: [README.md](README.md).
> punto banco 타블로 자동 진행 + 베팅 측(플레이어/뱅커/타이) 선택 + 칩 뱅크롤 정산 +
> 통산/연속 전적까지 UI에 연결됨(1인 vs 하우스, 로컬).

## 1. 개요

- **장르**: 1인 카지노 카드 게임(punto banco). 사람은 베팅 측·베팅액만 고르고, 양손(플레이어/뱅커)은
  타블로 규칙에 따라 자동으로 진행된다 — 사람이 카드 드로우를 조작하지 않는다.
- **덱**: 표준 52장 1덱([`card.ts`](../../src/domain/card.ts))을 매 판 셔플. 슈(여러 덱)·번 카드는 범위 밖.
- **목표**: 플레이어/뱅커 두 손 중 끗수(0~9)가 큰 쪽이 승, 같으면 타이. 사람은 그 결과에 베팅해
  가상 칩 뱅크롤을 불린다. 실거래·외부 인증·실제 베팅은 없다.

## 2. 규칙

### 끗수(끗) 계산 (`evaluateBaccaratHand`, `baccarat.ts`)
- 랭크 끗수값: **A=1**, 2~9=숫자 그대로, **10·J·Q·K=0**. 블랙잭과 달리 A는 항상 1(11로 올리지 않음).
- 손패 끗수 = 카드 끗수값 합을 **10으로 나눈 나머지**(0~9). 무늬(suit)는 끗수에 영향 없음.
- **내추럴**: 처음 2장만으로 끗수가 **8 또는 9**인 경우. 빈 손패는 throw(최소 1장 필요).

### 3rd-card 타블로 (`playBaccaratRound`, `playBaccaratRound.ts`)
- 분배: 플레이어 먼저 번갈아 2장씩(셔플 인덱스 플레이어 0·2 / 뱅커 1·3), 나머지는 드로우 큐.
- **내추럴 우선**: 어느 한쪽이라도 내추럴(8/9)이면 양쪽 모두 추가 드로우 없이 즉시 끗수 비교.
- **플레이어 규칙**: 초기 끗수 0~5면 세 번째 카드를 받고, 6~7이면 스탠드.
- **뱅커 규칙**:
  - 플레이어가 스탠드(6~7)면 → 뱅커는 0~5 드로우 / 6~7 스탠드.
  - 플레이어가 세 번째 카드를 받았으면 → 표준 타블로(플레이어 3rd 카드 끗수값 `p3` 기준):
    - 뱅커 0~2: 무조건 받음
    - 뱅커 3: `p3 ≠ 8`이면 받음
    - 뱅커 4: `p3 ∈ 2~7`이면 받음
    - 뱅커 5: `p3 ∈ 4~7`이면 받음
    - 뱅커 6: `p3 ∈ 6~7`이면 받음
    - 뱅커 7: 스탠드
- **승패**: 최종 끗수가 큰 쪽이 승(`player`/`banker`), 같으면 `tie`. 동일 RNG 시퀀스면 항상 동일 결과(결정적).
- 카드를 받아야 하는데 드로우 큐가 비면 throw(1덱 소진 방어).

### 베팅·배당 정산 (`settleBaccaratBet`, `baccarat.ts`)
- **플레이어 적중**: +bet (1:1).
- **뱅커 적중**: +`floor(bet × 0.95)` — 커미션 5% 차감(실수령 0.95:1), 내림으로 고정(단위 테스트로 보장).
- **타이 적중**: +bet × 8 (8:1).
- **타이 결과에서 플레이어/뱅커 베팅**: **푸시**(net 0, 원금 환원, 잔고 변화 없음).
- 그 외 미적중: −bet. bet이 양의 정수가 아니면 throw.

### A 싱글 / B 멀티 / C 전적

| 구분 | 상태 | 근거 |
| --- | --- | --- |
| **A. 싱글플레이** | ✅ | 끗수·타블로·정산 도메인 + 자동 진행 application + 베팅/뱅크롤/결과 UI. vs 하우스 1인 모드가 매끄럽게 동작. |
| **B. 멀티플레이(방 기반)** | ➖ **N/A** | 바카라는 사람이 베팅 측만 고르고 양손은 타블로로 자동 진행되는 **1인 vs 하우스** 게임이라, 두 플레이어가 턴을 주고받는 방 기반 원격 멀티가 의미 없다(ROADMAP DoD B 면제 규칙). 공용 테이블/멀티시트 베팅은 별도 기획 항목. |

## 4. UI/UX 요구사항

- [x] 목적·조작·배당 규칙 한 줄 안내(`.hint`) — 끗수 0~9·커미션 5%·타이 8:1·푸시 명시.
- [x] **시작 옵션**: 베팅 측(플레이어/뱅커/타이) + 베팅액(프리셋 10/50/100/500 또는 직접 입력) 선택.
- [x] 현재 베팅 측·베팅액·보유 칩(뱅크롤)을 `aria-live`로 표시.
- [x] 잔고 초과 베팅 차단(프리셋 비활성·입력 클램프), 베팅 불가 시 "딜링" 비활성.
- [x] 딜링 후 양손(플레이어/뱅커) 카드·끗수, **핸드 결과**(플레이어 승/뱅커 승/타이)와 **베팅 적중/빗나감/푸시**·정산 금액·갱신 잔고 표시.
- [x] 파산(칩 소진) 시 `role="alert"` 경고 + "새 뱅크롤로 리셋" 회복 경로.
