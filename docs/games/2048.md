# 2048 사양

> 규칙 + 구현 상태 + UI/UX 요구사항. 상위 인덱스: [README.md](README.md).

## 1. 개요

- **장르**: 슬라이드 퍼즐, 1인(플레이어 vs 자기 기록).
- **인원**: 1인.
- **목표**: 4×4 격자에서 타일을 한 방향으로 밀어 같은 숫자끼리 합쳐 **2048 타일**을 만든다.
  더 이상 움직일 수 없으면 게임 오버.

## 2. 규칙

- 행 우선(row-major) `board[row][col]` 4×4 격자. 빈 칸은 0.
- 한 수: 상/하/좌/우 한 방향으로 모든 타일을 끝까지 민다. 인접한 같은 값은 한 번 병합되며(한 이동에서 같은 타일이 두 번 병합되지 않음), 병합값이 점수로 누적된다.
- 이동으로 보드가 변했을 때만(`moved`) 빈 칸 하나에 새 타일(2: 90%, 4: 10%)을 무작위로 스폰한다. 막힌 방향은 보드 불변·스폰 없음.
- 새 타일 스폰의 무작위는 인프라 `RandomSource`(`MathRandomSource`) 주입으로 결정적 테스트가 가능하다(스폰 책임은 application).
- 승리: 보드에 목표(기본 2048) 이상 타일이 존재(`hasReachedTarget`).
- 게임 오버: 어느 방향으로도 이동(변화)이 불가능(`canMove === false`).

## 4. UI/UX 요구사항

- [x] 목적·조작 한 줄 안내(`.hint`): 타일을 밀어 합쳐 2048을 만든다.
- [x] 진행 상태 표시: 현재 점수(`gained` 합산)·최고 타일. 턴제가 아니므로 "차례" 대신 점수/진행을 노출.
- [x] 종료·승패 명확 구분(`.outcome`): 승리(목표 도달)와 게임오버(이동 불가)를 `describe2048Status`로 구분.
- [x] 잘못된 입력 피드백: 막힌 방향(보드 불변)은 `.error`로 안내.
- [x] 회복 경로: "새 게임" 버튼으로 `startGame` 재호출 리셋.
- [x] 키보드 조작: 화살표 키(`mapKeyToDirection`) + 실제 `<button>` 방향 컨트롤.
- [x] 색 비의존: 타일을 색뿐 아니라 숫자 텍스트로 구분(색각 이상 대응).
- [x] 좁은 화면 미파손: 보드 `max-width`·반응형 셀.
- [x] 구현 로직 실제 호출: 화면이 `startGame`/`play2048`를 호출(보기 전용 아님).
- [x] 기록 저장 + 화면 내 통산 전적·연승(`StreakPanel`+`selfStreakSummary`/`SELF_PLAYER`).

## 5. 알려진 갭 / 백로그

- **목표 도달 후 계속 진행(continue)**: 현재는 2048 도달을 종료(승)로 처리. 더 큰 타일을 노리는 continue 모드는 비목표(정의 필요).
- **실행 취소(undo)/이동 애니메이션**: 미구현(선택적 향상).
- **최고 점수 영속**: 화면 내 통산 전적은 저장되나 별도 best-score 영속은 범위 밖.
