# 바둑 (Go) 사양

> 규칙 + 구현 상태 + UI/UX 요구사항. 상위 인덱스: [README.md](README.md).
> 착수·따냄·패스·종료(연속 2패스)·영역 계가·승자까지 UI에 연결됨.

## 1. 개요

- **장르**: 2인 로컬(핫시트) 보드 게임.
- **보드**: 도메인 기본 19×19, **UI는 9×9** 고정.
- **목표**: 상대 돌을 따내고, 더 넓은 영역(집+돌)을 차지한다.

## 2. 규칙

### 착수·포획 (`go.ts`)
- 빈 칸에만 착수, 이미 돌/보드 밖이면 throw.
- 착수 순서: ① 돌 배치 → ② 활로(liberty) 0인 **인접 상대 그룹** 제거(포획) → ③ 포획 후에도
  자기 그룹의 활로가 0이면 **자살수**로 throw(보드 변형 없음).
- 활로 = 그룹에 4방향 인접한 빈 칸. 대각선은 인접이 아니다.
- **범위 밖(도메인 주석 명시)**: 패(ko) 반복 금지, 사석 판정.

### 계가 (`goScore.ts`)
- 영역(area) 계가: 각 색 점수 = 자기 돌 수 + 자기 집(빈 칸) 수.
- 빈 칸 영역을 4방향 flood-fill로 묶어, 경계가 **한 색에만** 닿으면 그 집, 양쪽/없음이면 중립(공배).
- `komi`(덤)를 백 점수에 더할 수 있다(기본 0).
- 점수 높은 쪽이 승자, 동점이면 무승부(`winner=null`).
- **가정**: 보드 위 모든 돌은 살아있다(사석 판정 없음).
