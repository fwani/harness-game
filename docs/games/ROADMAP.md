# 게임 완성 로드맵 — 한 번에 하나씩 (single source of truth)

> **이 파일이 자동화의 작업 대상을 정하는 단일 소스다.** planner·dev·qa 봇은 매 실행마다
> 이 파일을 읽고 **"현재 대상 게임" 1개에 수렴**해 그 게임을 완전히 끝낸 뒤 다음 번호로
> 넘어간다. 예전처럼 여러 게임을 조금씩 건드리지 않는다.

## 왜 이렇게 하나

게임이 많다 보니(약 50종) 플래너·QA가 매번 다른 게임을 조금씩 손봐서 어느 것도 "완성"에
도달하지 못했다. 그래서 **번호순으로 한 게임씩 완전히 완성**해 완성도를 실제로 끌어올린다.

## 현재 대상 게임

> **#1 baccarat (바카라)**

규칙: 아래 번호표에서 **상태가 ✅(완성)이 아닌 가장 작은 번호의 게임**이 현재 대상이다.
봇은 이 한 게임에만 작업을 집중한다. (이 줄도 게임이 완성될 때마다 함께 갱신한다.)

## 완성의 정의 (Definition of Done) — 모든 게임 공통

한 게임을 ✅(완성)으로 선언하려면 아래 **A·B·C와 문서·QA를 모두** 충족해야 한다.
게임별 세부 규칙은 그 게임의 위키/사양(`docs/games/<key>.md`, 없으면 인터넷 위키)을 기준으로 한다.

### A. 싱글플레이 완성
- [ ] 정식 규칙을 **위키 기준으로 완전 구현**(도메인 `src/domain/`) — 표준 규칙·예외·승패·무승부.
- [ ] 애플리케이션 오케스트레이터(`src/application/play*.ts`)로 한 판 진행이 가능.
- [ ] UI(`src/ui/games/<Game>.tsx`)에서 **끝까지 플레이 가능**: 조작 안내·현재 턴/상태·승패/무승부
      구분·불법수/오류 피드백·"새 게임" 리셋 경로.
- [ ] **시작 옵션 UI**(해당되면): 보드 크기·난이도·선공·변형 룰 등을 시작/방 화면에서 선택.
- [ ] vs CPU 또는 적절한 단일 모드가 매끄럽게 동작.
- [ ] 접근성/반응형: 키보드 조작, 색 비의존(기호+레이블), 좁은 화면(360px) 미파손.
- [ ] **디자인 토큰** 사용(생짜 hex 금지 — `docs/agent-harness/UX_GUIDELINES.md` "디자인 토큰" 참조).
- [ ] 데드코드 없음(구현된 로직이 화면에서 실제 호출).

### B. 멀티플레이(방 기반 원격) 완성
- [ ] `GameEngine` 어댑터로 코어가 통일돼 멀티 인프라가 이 게임을 지원
      (`src/infrastructure/server/`, `docs/agent-harness/exec-plans/active/game-server-multiplayer-ai.md` 단계).
- [ ] 방(room) 생성·접속, 턴 소유권, 같은 브라우저 여러 탭/다른 브라우저에서 같은 방 플레이.
- [ ] 시작 화면에서 **싱글(혼자/vs CPU) vs 멀티(방)** 모드 선택.
- (1인 퍼즐 등 멀티가 의미 없는 게임은 B를 "해당 없음(N/A)"으로 명시하고 면제 — 사유를 사양 문서에 기록.)

### C. 전적/유저 완성
- [ ] 판이 끝나면 `recordGame(...)`으로 결과 저장, 전적 화면에 노출.
- [ ] 전적이 **유저 단위**(익명 세션 id + 표시 이름)로 묶임. 싱글/멀티 전적 동일 유저 개념.

### 문서 · QA 게이트
- [ ] `docs/games/<key>.md` 사양 문서가 존재하고 "구현 상태" 매트릭스가 최신(없으면 신규 작성).
- [ ] QA 봇 플레이테스트 통과: 현재 이 게임에 열린 `qa-finding` 이슈가 없음, 콘솔 에러 없음.

## 완성 선언 → 다음 게임 전진 절차

1. planner가 현재 대상 게임의 DoD를 점검해, 미충족 항목을 **그 게임만의** 이슈로 만든다(아래 작업 규칙).
2. 모든 DoD가 충족되고 그 게임에 열린 이슈/`qa-finding`이 없으면, planner는 **"[완성] <게임> 완성
   확정 및 다음 게임 전진"** 이슈를 `ready-for-dev`로 1건 만든다(이미 열려 있으면 만들지 않음).
3. dev가 그 이슈를 처리: 최종 검증(`sh scripts/agent-harness/agent-verify`) 후, 이 파일에서 해당
   게임의 **상태를 ✅로 바꾸고 "현재 대상 게임" 줄을 다음 번호로 갱신**, 사양 문서 매트릭스도 갱신.
4. 다음 실행부터 봇들은 자동으로 다음 번호 게임에 수렴한다.

## 완성 대상 아님 (도구/메타)

아래는 단일 게임이 아니라 도구/메타 기능이라 번호 로드맵에서 제외한다. 필요한 개선은
일반 이슈로 처리하되, 위 게임 완성 흐름을 막지 않는다.

- `deal` 카드 딜(카드 분배 유틸) · `ladder` 사다리타기 · `selfplay` 관전(자동 대전)
- `tournament` 토너먼트 · `knockout` 녹아웃 · `records` 전적(뷰어)
